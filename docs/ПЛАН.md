## ЧТО ДЕЛАЕМ ДАЛЬШЕ (по порядку)

### ЭТАП 1 — Запускаем core-svc и проверяем авторизацию
**Статус: следующий шаг**

Что нужно сделать в терминале:
1. Скопировать новую схему: `cp scripts/schema_core_NEW.prisma services/core-svc/prisma/schema.prisma`
2. Установить пакеты (JWT, bcrypt, passport)
3. Запустить миграцию Prisma → создадутся все таблицы в `core_db`
4. Запустить core-svc и проверить через Postman:
   - POST `/auth/register`
   - POST `/auth/login`
   - GET `/auth/me`
   - GET `/auth/google`

## АРХИТЕКТУРА (highload)

```
Фронтенд (Next.js / React)
        ↓
   Gateway :3000        ← единая точка входа
   /api/core → :3001    ← авторизация, организации, новости, гайды
   /api/ai   → :8000    ← AI чат, голос
   /api/taxi → :3002    ← такси (позже)
        ↓
  PostgreSQL :5432       ← core_db, taxi_db, ai_db
  Redis :6379            ← сессии, кэш
```

**Highload защита:**
- Каждый сервис независимый — если AI упал, авторизация работает
- Redis для кэширования частых запросов
- Pagination везде (не грузим сразу 1000 организаций)
- JWT stateless — не нагружает БД при каждом запросе
